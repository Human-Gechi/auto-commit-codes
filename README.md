Project details
- `fetch_github_metrics` function : Sends a request to see if a connection to the repo can be made.
- `check_file_exists` : Checks if the file exists by obtainning the SHA from the jon response
- `get_blob_sha` : Assigns a she64 digest to mimick one generated by github inorder to assertain the existence of a file by comparing the SHA keys
- `upload_queries_to_github` : This function dos the uploading to the repo. It checks if the file exists and skips the file and if it eists but an update has been made, it makes the updates and commits the updates to the repo.. If the file has never been created, a new file iscreated and a commit is made to the repo.
`uploaded_count` : This counts the number of files in the repo that were uploaded as in : New files
`updated_count` : This counts the number of files in the repo that were updated
`failed_count` : This records the count for failed uploads maybe due to network issues ir an invalid SHA key
`skipped_count` : Counts files that were skipped because they existin the repo and no changes were made.
